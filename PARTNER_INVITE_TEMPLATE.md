## Screen Share Session Script

**If partner wants live walkthrough:**

### Preparation (Before call)
- [ ] Orchestrator running
- [ ] ngrok tunnel active
- [ ] URL tested and working
- [ ] Have PARTNER_TESTING_GUIDE.md open

### During Call (15-30 min)

**Intro (2 min):**
"Thanks for doing this! I'm going to have you share your screen and just use it naturally. I'll only jump in if you're totally stuck. I want to see where you get confused - that's the valuable feedback!"

**Instructions:**
1. "Open this link: [paste URL]"
2. "Just explore and tell me what you're thinking out loud"
3. Let them read welcome screen - DON'T NARRATE
4. Only help if they're stuck for >30 seconds

**Questions to ask during (don't interrupt, ask when natural):**
- "What do you think will happen when you click that?"
- "Is this what you expected?"
- "What would make this clearer?"
- "What's missing here?"

**After project creation:**
- "Does the progress make sense to you?"
- "Do you know how long this will take?"
- "Can you explain what's happening in each phase?"

**After completion:**
- "Is this what you expected to receive?"
- "What would you do with this code?"
- "Do you feel confident this would actually work?"

**Wrap-up (5 min):**
- "What worked well?"
- "What was frustrating or confusing?"
- "What's missing that you expected?"
- "Would you use this for real projects?"
- "If this saved you 10+ hours, what would you pay per MVP?"

### Post-Call
- [ ] Send thank you message
- [ ] Share compensation (discount code, beta access, etc.)
- [ ] Document feedback in testing notes
- [ ] Create issues for bugs/improvements
- [ ] Share roadmap if they're interested in following progress
